Vim 편집기 기초 가이드

대상: 터미널에서 파일을 수정해야 하는 모든 개발자 목표: 서버에 붙어서 긴급 수정할 때 멘붕 없이 Vim 사용하기

왜 Vim을 알아야 하나?
1. 서버에는 VS Code가 없다
SSH로 서버에 접속하면 터미널만 있음
긴급 수정 시 Vim이 유일한 선택지인 경우 많음
2. 대부분의 서버에 기본 설치
Linux, macOS에 기본 포함
별도 설치 없이 바로 사용 가능
3. 바이브코딩에서 자주 마주침
.env
클릭하여 복사
 파일 수정
설정 파일 긴급 패치
로그 확인 중 수정 필요할 때
4. Git 커밋 메시지 작성
git commit
클릭하여 복사
 시 에디터로 Vim이 열림
모르면 빠져나오지 못함 😱
1️⃣ 가장 중요한 것: 탈출 방법
🚨 Vim에서 나가는 법 (외워라!)
:q        # 나가기 (변경 없을 때)
:q!       # 강제 나가기 (변경 무시)
:wq       # 저장하고 나가기
:x        # 저장하고 나가기 (wq와 같음)

2️⃣ Vim의 모드 이해 (핵심 개념)

Vim은 모드 기반 편집기다.

모드	설명	진입 방법
Normal	이동, 명령 실행	ESC
클릭하여 복사

Insert	텍스트 입력	i
클릭하여 복사
, a
클릭하여 복사
, o
클릭하여 복사

Visual	텍스트 선택	v
클릭하여 복사

Command	저장, 종료 등	:
클릭하여 복사
모드 전환 흐름
Normal ──(i)──> Insert ──(ESC)──> Normal
   │
   └──(:)──> Command ──(Enter)──> Normal

1️⃣4️⃣ 필수 명령어 치트시트 (복사용)

:w      저장
:q      종료
:wq     저장 + 종료
:q!     강제 종료 (저장 안 함)

# 이동
gg      맨 위

# 검색

n       다음
N       이전

# 치환
:%s/old/new/g   전체 바꾸기
