속성 셀렉터
/* 속성 존재 */
[disabled] { }       /* disabled 속성이 있는 모든 요소 */
input[required] { }  /* required 속성이 있는 input */

/* 정확한 값 */
[type="text"] { }
input[type="email"] { }
[data-status="active"] { }

/* 값 시작 (^=) */
a[href^="https"] { }    /* https로 시작하는 링크 */
[class^="btn-"] { }     /* btn-으로 시작하는 클래스 */

/* 값 끝 ($=) */
a[href$=".pdf"] { }     /* .pdf로 끝나는 링크 */
img[src$=".png"] { }

/* 값 포함 (*=) */
[class*="container"] { } /* container를 포함하는 클래스 */
a[href*="google"] { }

/* 단어 포함 (~=) - 공백으로 구분된 단어 */
[class~="active"] { }   /* class="btn active" 매칭 */

/* 하이픈 구분 시작 (|=) */
[lang|="en"] { }        /* en 또는 en-US, en-GB 등 */

7.2 쿠키 읽기
// 모든 쿠키 문자열
console.log(document.cookie);
// "username=kim; theme=dark; token=abc123"

// 특정 쿠키 값 가져오기
function getCookie(name) {
  const value = `; ${document.cookie}`;
  const parts = value.split(`; ${name}=`);
  if (parts.length === 2) {
    return parts.pop().split(';').shift();
  }
  return null;
}

console.log(getCookie('username')); // "kim"

7.5 SameSite 옵션
SameSite=Strict
- 같은 사이트에서만 전송
- 외부 링크로 접근 시 쿠키 없음

SameSite=Lax (기본값)
- 탑 레벨 내비게이션에서만 전송
- 링크 클릭은 OK, 폼 POST나 iframe은 X

SameSite=None; Secure
- 크로스 사이트에서도 전송
- 반드시 Secure와 함께 사용

7.7 세션 vs 쿠키
브라우저 세션:
- sessionStorage 또는 쿠키(만료일 없음)
- 탭/브라우저 닫으면 삭제

서버 세션:
- 서버에서 세션 ID 생성
- 클라이언트에 세션 ID를 쿠키로 전달
- 서버에서 세션 데이터 관리
- 장점: 보안 (데이터가 서버에)
- 단점: 서버 메모리/저장소 필요

JWT 토큰:
- 서버가 서명한 토큰을 클라이언트가 보관
- localStorage 또는 쿠키에 저장
- 장점: 서버 무상태, 확장성
- 단점: 크기가 큼, 만료 전 무효화 어려움
