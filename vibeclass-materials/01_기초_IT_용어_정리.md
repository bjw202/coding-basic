바이브코딩을 위한 기초 IT 용어 정리

대상: 바이브코딩( AI와 함께 코딩 )을 처음부터 제대로 이해하려는 사람 목표: 용어 때문에 막히지 않게, AI에게 정확히 지시할 수 있는 최소 개념 정리

1️⃣ 컴퓨터 / 시스템 기본
운영체제(OS)
컴퓨터 하드웨어를 관리하고 프로그램을 실행해주는 기본 소프트웨어
예: Windows, macOS, Linux(우분투)
프로세스(Process)
실행 중인 프로그램 하나
서버에서 "프로세스가 죽었다" = 프로그램이 종료됨
스레드(Thread)
프로세스 안에서 실제 작업을 수행하는 실행 단위
여러 작업을 동시에 처리할 때 사용
메모리(RAM)
실행 중인 프로그램이 사용하는 작업 공간
부족하면 느려지거나 종료됨
CPU(중앙처리장치)
컴퓨터의 두뇌
모든 연산과 명령을 처리하는 핵심 부품
GPU(그래픽처리장치)
그래픽 연산 전문 프로세서
AI/딥러닝 학습에도 많이 사용
SSD / HDD
데이터를 영구 저장하는 저장 장치
SSD가 HDD보다 빠름
커널(Kernel)
운영체제의 핵심
하드웨어와 소프트웨어 사이를 연결
드라이버(Driver)
하드웨어를 운영체제가 인식하게 해주는 프로그램
예: 프린터 드라이버, 그래픽 드라이버
가상 메모리
RAM이 부족할 때 디스크를 임시 메모리처럼 사용
느려지는 원인 중 하나
부팅(Booting)
컴퓨터를 켜서 운영체제가 시작되는 과정
BIOS/UEFI
컴퓨터 시작 시 하드웨어를 초기화하는 펌웨어
부팅 순서 등 설정
펌웨어(Firmware)
하드웨어에 내장된 소프트웨어
제조사가 미리 설치
멀티태스킹(Multitasking)
여러 프로그램을 동시에 실행
OS가 관리
데몬(Daemon)
백그라운드에서 실행되는 서비스 프로그램
예: 웹 서버, DB 서버
서비스(Service)
백그라운드에서 동작하는 프로그램
Windows에서의 데몬
힙(Heap)
동적으로 할당되는 메모리 영역
프로그램 실행 중 크기 변경 가능
스택(Stack)
함수 호출 시 사용되는 메모리 영역
후입선출(LIFO) 구조
버퍼(Buffer)
데이터를 임시로 저장하는 공간
처리 속도 차이를 해결
캐시 메모리
CPU와 RAM 사이의 빠른 임시 저장소
자주 쓰는 데이터 저장
64비트 / 32비트
CPU가 한 번에 처리하는 데이터 크기
64비트가 더 큰 메모리 사용 가능
바이트(Byte)
데이터 크기 단위
1KB = 1024 Bytes
이진수(Binary)
0과 1로만 표현하는 숫자 체계
컴퓨터의 기본 언어
16진수(Hexadecimal)
0-9, A-F로 표현하는 숫자 체계
색상 코드(#FF0000)에 사용
2️⃣ 네트워크 / 웹 기초
IP 주소
인터넷에서 컴퓨터의 주소
예: 123.45.67.89
클릭하여 복사
IPv4 / IPv6
IP 주소 버전
IPv4: 32비트, IPv6: 128비트
포트(Port)
하나의 컴퓨터에서 프로그램을 구분하는 번호
예: 웹 서버는 보통 80, 443, 개발 서버는 3000
HTTP / HTTPS
웹에서 데이터를 주고받는 규칙(프로토콜)
HTTPS는 암호화된 HTTP
요청(Request) / 응답(Response)
클라이언트가 요청 → 서버가 응답
GET / POST / PUT / DELETE
HTTP 메서드들
조회/생성/수정/삭제
헤더(Header)
HTTP 요청/응답의 메타데이터
인증 정보, 콘텐츠 타입 등
바디(Body)
HTTP 요청/응답의 실제 데이터
JSON, HTML 등
상태 코드(Status Code)
HTTP 응답 결과를 나타내는 숫자
200(성공), 404(없음), 500(서버 에러)
DNS(Domain Name System)
도메인 이름을 IP 주소로 바꿔주는 시스템
google.com
클릭하여 복사
 → 142.250.196.110
클릭하여 복사
도메인(Domain)
사람이 읽기 쉬운 웹사이트 주소
예: vibeclass.kr
클릭하여 복사
서브도메인(Subdomain)
도메인 앞에 붙는 추가 주소
예: api.vibeclass.kr
클릭하여 복사
URL(Uniform Resource Locator)
웹 자원의 전체 주소
예: https://vibeclass.kr/courses/1
클릭하여 복사
URI(Uniform Resource Identifier)
자원을 식별하는 문자열
URL보다 더 넓은 개념
프로토콜(Protocol)
데이터를 주고받는 약속된 규칙
예: HTTP, FTP, SSH, WebSocket
TCP/IP
인터넷 통신의 기본 규칙
데이터를 패킷으로 나눠서 전송
UDP
빠르지만 신뢰성 없는 전송 프로토콜
실시간 스트리밍에 사용
패킷(Packet)
네트워크로 전송되는 데이터 조각
헤더 + 데이터로 구성
클라이언트(Client)
서비스를 요청하는 쪽
예: 웹 브라우저, 앱
서버(Server)
서비스를 제공하는 쪽
예: 웹 서버, 데이터베이스 서버
로컬호스트(localhost)
내 컴퓨터 자신을 가리키는 주소
127.0.0.1
클릭하여 복사
 또는 localhost
클릭하여 복사
방화벽(Firewall)
네트워크 보안 장치
허용/차단 규칙으로 트래픽 제어
SSL/TLS
데이터를 암호화하는 보안 프로토콜
HTTPS의 핵심 기술
레이턴시(Latency)
요청부터 응답까지 걸리는 시간
"지연 시간"이라고도 함
쓰루풋(Throughput)
단위 시간당 처리량
초당 요청 수 등
대역폭(Bandwidth)
한 번에 전송할 수 있는 데이터 양
넓을수록 빠름
CDN(Content Delivery Network)
전 세계에 콘텐츠를 분산 저장해서 빠르게 전달
이미지, 동영상 등에 사용
WebSocket
서버와 클라이언트가 실시간으로 양방향 통신
채팅, 실시간 알림에 사용
웹훅(Webhook)
이벤트 발생 시 자동으로 알림을 보내는 방식
결제 완료 알림 등
롱폴링(Long Polling)
서버가 응답을 늦게까지 유지
WebSocket 이전 실시간 통신 방식
SSE(Server-Sent Events)
서버에서 클라이언트로 단방향 실시간 데이터 전송
주식 시세, 알림 등
AJAX(Asynchronous JavaScript and XML)
페이지 새로고침 없이 서버와 통신
현대 웹의 기본
Fetch API
브라우저의 HTTP 요청 API
AJAX의 현대 버전
axios
인기 있는 HTTP 클라이언트 라이브러리
Promise 기반
VPN(Virtual Private Network)
가상 사설망
보안 통신, 지역 우회
프록시(Proxy)
클라이언트와 서버 사이 중개 역할
게이트웨이(Gateway)
서로 다른 네트워크를 연결하는 장치
라우터(Router)
네트워크 경로를 결정하는 장치
스위치(Switch)
네트워크 장치들을 연결
LAN / WAN
근거리 통신망 / 광역 통신망
NAT(Network Address Translation)
사설 IP를 공인 IP로 변환
공유기에서 사용
DHCP
IP 주소 자동 할당 프로토콜

4️⃣ 데이터 / DB 기초

DBMS(Database Management System)
데이터베이스 관리 시스템

페이징에 사용

🎯 바이브코딩 핵심 한 줄 요약
코딩은 "문법"보다 개념
AI에게 시키려면 용어를 정확히 써야 한다
용어를 알면, 바이브코딩 성공 확률이 급상승
범위 지정 + 정확한 용어 = 사고 없는 AI 코딩

이 문서는 바이브코딩의 사전(辭典) 이다. 막히면 다시 여기로 돌아와라.
