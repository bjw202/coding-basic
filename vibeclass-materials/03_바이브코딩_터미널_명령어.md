바이브코딩용 터미널 필수 명령 문서

목표: ChatGPT/코파일럿/커서(CLI)로 코드 짜고 → 실행/디버그 → Git 커밋/배포까지 ‘바이브코딩’에서 가장 많이 쓰는 터미널 명령만 복붙용으로 정리

0) 루틴 5개 (매번 하는 것)
pwd
ls
cd 프로젝트
code .   # VSCode (설치돼 있으면)

복사
1) 프로젝트 생성/실행 (Node/JS)
Node 버전 확인
node -v
npm -v

복사
의존성 설치
npm i
# 또는
pnpm i
# 또는
yarn

복사
개발 서버 실행
npm run dev
npm start

복사
빌드/테스트
npm run build
npm test

복사
포트 점유 확인/해제 (맥)
lsof -i :3000
kill -9 PID

복사
2) Git (바이브코딩 핵심)
현재 상태 확인
git status

복사
변경 확인
git diff

복사
스테이징/커밋
git add .
git commit -m "feat: add login flow"

복사
브랜치
git branch
git switch -c feat/login
# (구버전)
git checkout -b feat/login

복사
최신 가져오기/푸시
git pull
git push -u origin feat/login

복사
실수 복구 TOP
# 스테이징 취소
git restore --staged .

# 파일 변경 되돌리기(주의)
git restore .

# 마지막 커밋 메시지 수정
git commit --amend

# 마지막 커밋 취소(변경은 남김)
git reset --soft HEAD~1

복사
3) 로그/디버깅 (문제 생기면 무조건)
로그 보기
cat log.txt
less log.txt            # q로 종료

복사
실시간 로그
tail -f log.txt

복사
에러 검색
grep -n "error" -R .

복사
프로세스/강제종료
ps aux | grep node
kill -9 PID

복사
4) 파일/폴더 (자동화·정리)
생성/복사/이동/삭제
mkdir tmp
cp -R src backup_src
mv a.txt b.txt
rm -rf tmp              # ⚠️ 조심

복사
찾기
find . -name "*.ts"
find . -maxdepth 2 -type f -name "*.env*"

복사
용량 확인 (맥)
du -sh .
du -sh node_modules

복사
5) 환경변수 / .env
임시 환경변수
export NODE_ENV=production
export PORT=3000

복사
확인
echo $NODE_ENV

6) Python (AI/자동화 자주 쓰면)
가상환경

python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

복사
종료
deactivate

7) Docker (배포/실행 환경 통일)

docker run --rm -p 3000:3000 app:dev

복사
컨테이너/이미지 정리
docker ps
docker images
docker system prune -f

복사
8) SSH/서버 (EC2에 붙어서 작업)
접속

복사
