claude.md

이 문서는 Claude Code / Cursor / Copilot 등 AI 코딩 도구가 이 레포지토리에서 따라야 할 절대 규칙을 정의한다. 목적: 빠르게 개발하되, 사고 없이 · 계약을 깨지 않고 · 운영 안정성을 유지하는 것.

1. 기본 원칙 (절대 규칙)
AI는 보조 도구이며, 최종 책임은 사람에게 있다.
범위 지정 없는 수정은 금지한다.
한 번의 요청은 하나의 목적만 가진다.
기존 동작 / API 계약 / 데이터 구조를 암묵적으로 변경하지 않는다.
2. 작업 범위 규칙

AI는 아래 규칙을 반드시 따른다.

수정 범위는 명시된 디렉토리/파일만 허용된다.
명시되지 않은 파일은 읽기만 가능하다.
node_modules
클릭하여 복사
, dist
클릭하여 복사
, build
클릭하여 복사
 디렉토리는 항상 무시한다.

예시 프롬프트

src/api 디렉토리만 수정하고, 다른 파일은 절대 변경하지 마라.

복사
3. API 계약 규칙 (가장 중요)
기존 API 요청/응답 스키마는 절대 변경 금지.
변경이 필요한 경우:
새로운 버전(v2)으로 분리
혹은 하위 호환성을 유지

AI에게 반드시 명시할 문장

기존 API 응답 스키마는 절대 변경하지 말 것.

복사
4. 에러 처리 규칙
모든 실패는 의미 있는 HTTP 상태 코드를 사용한다.
성공/실패를 200 하나로 처리하지 않는다.
에러 응답 형식은 아래로 통일한다.
{ "error": { "code": "ERROR_CODE", "message": "human readable message" } }

복사
5. 인증 / 보안 규칙
인증/인가 로직은 AI 단독 수정 금지.
토큰, 비밀번호, 개인정보는:
로그에 남기지 않는다.
코드에 하드코딩하지 않는다.
6. 로깅 / 관측성 규칙
운영 로그 제거 금지.
에러 로그에는 최소한 아래 정보 포함:
requestId
사용자 식별자(가능한 경우)
핵심 파라미터(민감 정보 제외)
7. 테스트 규칙
로직 변경 시 테스트 없는 수정 금지.
버그 수정은 반드시 재현 테스트 → 수정 → 통과 순서로 진행.
테스트가 불가능한 경우, 그 이유를 명시한다.
8. 성능 / 안정성 규칙
성능 개선 요청 시:
가정한 트래픽 규모를 명시한다.
병목 지점을 설명한다.
멱등성이 필요한 API는 반드시 보장한다.
9. Git / 협업 규칙
AI가 수정한 PR은 제목에 [AI]
클릭하여 복사
를 붙인다.
PR은 1 기능 = 1 PR 원칙을 따른다.
대규모 변경은 단계적으로 PR을 분리한다.
10. 배포 / 운영 안전 규칙
AI 코드의 자동 배포는 금지한다.
모든 변경은 롤백 가능해야 한다.
배포 전 반드시 다음을 확인한다:
테스트 통과
API 계약 유지
로그 정상 출력

전체 코드 정리해줘

알아서 최적화해줘

[검증]

- 중복 요청 재현 테스트 추가
