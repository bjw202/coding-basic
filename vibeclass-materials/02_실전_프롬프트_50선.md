Claude Code 실전 프롬프트 50선

목적: 바이브코딩 / 실무에서 바로 복붙해서 쓰는 프롬프트 모음
원칙: 짧게 · 명확하게 · 범위 좁게

1️⃣ 프로젝트 이해 (1~5)
이 프로젝트 전체 구조를 설명해줘
주요 엔트리 포인트가 어디인지 알려줘
이 레포에서 가장 중요한 비즈니스 로직은 뭐야?
폴더 구조가 왜 이렇게 설계됐는지 설명해줘
이 프로젝트의 실행 흐름을 단계별로 설명해줘
2️⃣ 코드 설명 / 분석 (6~10)
이 파일이 무슨 역할을 하는지 설명해줘
이 함수의 입력과 출력, 책임을 정리해줘
이 코드에서 사이드 이펙트가 발생하는 부분을 알려줘
유지보수 관점에서 위험한 코드가 있는지 찾아줘
이 코드의 시간복잡도/성능 이슈가 있을지 분석해줘
3️⃣ 리팩토링 (11~15)
이 코드를 더 읽기 쉽게 리팩토링해줘
중복되는 로직을 함수로 분리해줘
네이밍을 더 명확하게 바꿔줘
주석 없이도 이해되게 코드 구조를 개선해줘
기존 동작은 유지하면서 구조만 정리해줘
4️⃣ 기능 추가 (16~20)
이 API에 페이지네이션을 추가해줘
JWT 기반 로그인 기능을 추가해줘
입력값 검증 로직을 추가해줘
실패 시 에러 응답을 통일해줘
옵션 파라미터를 받을 수 있게 확장해줘
5️⃣ 버그 수정 / 디버깅 (21~25)
이 코드에서 발생 가능한 버그를 찾아줘
아래 에러 로그를 기준으로 문제를 해결해줘
특정 조건에서만 발생하는 버그를 재현해줘
null/undefined 관련 예외를 방어해줘
경계값(빈 값, 최대값) 처리 로직을 보완해줘
6️⃣ 테스트 (26~30)
이 파일에 대한 유닛 테스트를 작성해줘
중요한 케이스 위주로 테스트를 추가해줘
실패하는 테스트를 통과하게 수정해줘
테스트 가독성을 개선해줘
이 로직을 테스트하기 쉽게 구조를 바꿔줘
7️⃣ 멀티파일 작업 (31~35)
이 함수가 사용되는 모든 위치를 찾아줘
관련된 파일들을 함께 수정해줘
이 변경이 영향을 주는 범위를 정리해줘
공통 모듈로 분리하는 게 좋을지 판단해줘
레거시 코드와의 호환성을 유지해줘
8️⃣ 성능 / 안정성 (36~40)
성능 병목이 될 수 있는 부분을 찾아줘
불필요한 연산이나 호출을 제거해줘
동시성/경쟁 상태 문제가 있을지 점검해줘
메모리 누수 가능성을 분석해줘
운영 환경에서 안전하게 동작하게 보완해줘
9️⃣ Git / 협업 (41~45)
이번 변경 내용을 한 문단으로 요약해줘
컨벤션에 맞는 커밋 메시지를 만들어줘
PR 설명을 작성해줘
리뷰어가 봐야 할 포인트를 정리해줘
롤백 시 주의할 점을 알려줘
🔟 바이브코딩 제어 문장 (46~50)
설명은 하지 말고 코드만 수정해줘
변경은 최소한으로 해줘
기존 코드 스타일을 그대로 따라줘
production 환경 기준으로 고려해줘
엣지 케이스까지 반드시 처리해줘
✅ 사용 팁
항상 범위 제한 문장과 함께 쓰기
큰 작업은 1~2개 프롬프트씩 나눠서 요청
자동 수정 후 git diff
클릭하여 복사
로 직접 검증
