## 2. 변수 선언

```javascript
// var (구식, 사용 비권장)
var name = "홍길동";

// let (재할당 가능)
let age = 25;
age = 26;  // OK

// const (재할당 불가, 권장)
const PI = 3.14159;
// PI = 3.14;  // Error!

// const로 선언한 객체/배열의 내용은 수정 가능
const user = { name: "홍길동" };
user.name = "김철수";  // OK
```

> **실무 규칙**: `const`를 기본으로 사용하고, 재할당이 필요할 때만 `let`을 사용합니다. `var`는 사용하지 않습니다.

## 3. 자료형

```javascript
// 원시 타입
const str = "문자열";           // String
const num = 42;                 // Number
const bool = true;              // Boolean
const empty = null;             // Null
const notDefined = undefined;   // Undefined

// 타입 확인
console.log(typeof str);        // "string"
console.log(typeof num);        // "number"
console.log(typeof null);       // "object" (역사적 버그)

// 타입 변환
console.log(Number("123"));     // 123
console.log(String(123));       // "123"
console.log(Boolean(0));        // false
console.log(Boolean("hello"));  // true
```

## 4. 비교와 논리 연산

```javascript
// === 를 사용하세요 (값과 타입 모두 비교)
console.log(5 == "5");   // true  (권장하지 않음)
console.log(5 === "5");  // false (권장)

// Nullish Coalescing (??)
const value = null;
console.log(value ?? "기본값");  // "기본값"
console.log(0 ?? "기본값");      // 0 (0은 null이 아님)

// Optional Chaining (?.)
const user = { name: "홍길동" };
console.log(user?.address?.city);  // undefined (에러 없음)
```

## 5. 배열 (Array)

```javascript
const fruits = ["사과", "바나나", "오렌지"];

// 기본 메서드
fruits.push("포도");        // 끝에 추가
fruits.pop();               // 마지막 제거
fruits.unshift("키위");     // 앞에 추가
fruits.shift();             // 첫 번째 제거

// 고차 함수 (실무에서 많이 사용)
const numbers = [1, 2, 3, 4, 5];

const doubled = numbers.map(n => n * 2);        // [2, 4, 6, 8, 10]
const evens = numbers.filter(n => n % 2 === 0); // [2, 4]
const sum = numbers.reduce((acc, n) => acc + n, 0); // 15

// 배열 검색
console.log(numbers.find(n => n > 3));    // 4 (첫 번째 매칭)
console.log(numbers.some(n => n > 4));    // true
console.log(numbers.every(n => n > 0));   // true
```

## 6. 객체 (Object)

```javascript
const user = {
  name: "홍길동",
  age: 30,
  greet() {
    return `안녕하세요, ${this.name}입니다`;
  }
};

// 비구조화 할당 (실무에서 매우 많이 사용)
const { name, age } = user;

// 스프레드 연산자
const updatedUser = { ...user, age: 31 };

// 단축 속성명
const x = 1, y = 2;
const point = { x, y };  // { x: 1, y: 2 }
```

## 7. 함수

```javascript
// 일반 함수
function add(a, b) {
  return a + b;
}

// 화살표 함수 (실무 표준)
const multiply = (a, b) => a * b;

// 기본값 매개변수
const greet = (name = "사용자") => `안녕하세요, ${name}!`;

// 나머지 매개변수
const sum = (...numbers) => numbers.reduce((acc, n) => acc + n, 0);
sum(1, 2, 3, 4, 5); // 15
```

## 8. 비동기 프로그래밍

```javascript
// Promise 기반
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));

// async/await (더 읽기 쉬운 방식, 권장)
async function getUsers() {
  try {
    const response = await fetch('https://api.example.com/users');
    const data = await response.json();
    return data;
  } catch (error) {
    console.error('오류:', error);
    throw error;
  }
}

// 병렬 처리
async function getMultipleData() {
  const [users, products] = await Promise.all([
    fetch('/api/users').then(r => r.json()),
    fetch('/api/products').then(r => r.json())
  ]);
  return { users, products };
}
```

> **Session 2 연결**: Node.js와 Express에서 데이터베이스 조회, API 호출 등 모든 I/O 작업이 비동기로 처리됩니다. async/await 패턴을 완전히 익히는 것이 중요합니다.

## 9. 클래스 (ES6+)

```javascript
class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    return `${this.name}이 소리를 냅니다`;
  }
}

class Dog extends Animal {
  speak() {
    return `${this.name}이 짖습니다: 왈왈!`;
  }
}

const dog = new Dog("바둑이");
console.log(dog.speak()); // "바둑이이 짖습니다: 왈왈!"
```

## 핵심 포인트 정리

- `const`를 기본으로 사용하고, 재할당이 필요할 때만 `let`을 사용하세요
- `===`(엄격 비교)를 `==` 대신 항상 사용하세요
- `map`, `filter`, `reduce`는 실무에서 가장 많이 쓰는 배열 메서드입니다
- async/await 패턴을 익히면 Promise 체인보다 훨씬 읽기 쉬운 코드를 작성할 수 있습니다
- `?.`(옵셔널 체이닝)과 `??`(널리시 병합)은 실무에서 undefined/null 오류를 막는 핵심 도구입니다
