# Node.js 핵심

## 학습 목표

이 자료를 마치면 다음을 할 수 있습니다:
- Node.js가 어떻게 동작하는지 원리를 이해할 수 있습니다

- 파일 시스템, HTTP, path 등 내장 모듈을 활용할 수 있습니다
- npm을 사용해 패키지를 관리할 수 있습니다

---

## 1. Node.js란?

JavaScript를 서버에서 실행할 수 있게 해주는 런타임 환경입니다.

- V8 엔진 (Chrome의 JavaScript 엔진)
- libuv (비동기 I/O 라이브러리)
- 내장 API (fs, http, path 등)

### 왜 Node.js인가?

| 장점 | 설명 |
|------|------|
| JavaScript 통합 | 프론트와 백엔드 모두 JS로 개발 |
| 비동기 I/O | 높은 동시 처리 성능 |
| 거대한 생태계 | npm에 200만+ 패키지 |
| 실시간 처리 | WebSocket, SSE 등에 적합 |

### Node.js vs 브라우저

| 항목 | Node.js | 브라우저 |
|------|---------|---------|
| 실행 환경 | 서버 (터미널) | 클라이언트 (브라우저) |
| `window` | 없음 | 있음 |
| `document` | 없음 | 있음 |
| `fs` (파일) | 있음 | 없음 |
| `process` | 있음 | 없음 |
| `fetch` | 있음 (v18+) | 있음 |

## 3. 이벤트 루프

Node.js가 싱글 스레드임에도 많은 요청을 처리할 수 있는 이유입니다.

```
Call Stack
    ↓
Web APIs (setTimeout, fs.read, http.request)
    ↓
Event Queue
    ↓
Event Loop → Call Stack이 비면 Queue에서 꺼냄
```

```javascript
console.log('1. 시작');

setTimeout(() => {
    console.log('2. setTimeout (0ms)');
}, 0);

Promise.resolve().then(() => {
    console.log('3. Promise');
});

console.log('4. 끝');

// 출력 순서: 1 → 4 → 3 → 2
// Promise는 마이크로태스크로 setTimeout보다 먼저 실행됩니다
```

## 4. 파일 시스템 (fs 모듈)

```javascript
import { readFile, writeFile, readdir } from 'fs/promises';

// 파일 읽기
const content = await readFile('data.txt', 'utf8');
console.log(content);

// 파일 쓰기
await writeFile('output.txt', '내용입니다', 'utf8');

// 폴더 내 파일 목록
const files = await readdir('./src');
console.log(files);
```

import path from 'path';

// 현재 파일 기준 경로 (ES 모듈)
import { fileURLToPath } from 'url';

## 6. HTTP 모듈

```javascript
import http from 'http';

const server = http.createServer((req, res) => {
    if (req.url === '/') {
        res.writeHead(200, { 'Content-Type': 'text/html; charset=utf-8' });
        res.end('<h1>Hello, Node.js!</h1>');
    } else if (req.url === '/api/data') {
        res.writeHead(200, { 'Content-Type': 'application/json' });
        res.end(JSON.stringify({ message: '안녕하세요!' }));
    } else {
        res.writeHead(404);
        res.end('Not Found');
    }
});

server.listen(3000, () => {
    console.log('서버 실행 중: http://localhost:3000');
});
```

> **실무에서는**: http 모듈을 직접 쓰기보다 Express 같은 프레임워크를 사용합니다. http 모듈의 원리를 이해하면 Express의 동작을 더 잘 이해할 수 있습니다.

## 핵심 포인트 정리

- Node.js는 싱글 스레드지만 이벤트 루프 덕분에 많은 요청을 처리할 수 있습니다
- `fs`, `path`, `http` 모듈은 Node.js 내장 모듈이라 설치 없이 사용 가능합니다
- 환경변수는 `process.env`로 접근하고, `.env` 파일과 `dotenv` 패키지를 함께 사용합니다
- `npm install -D` 는 개발 도구를, `-D` 없이는 운영 의존성을 추가합니다
