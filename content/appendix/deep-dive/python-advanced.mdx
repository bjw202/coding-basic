## 1. 비동기 프로그래밍 (asyncio)

### 동기 vs 비동기

```
동기:  작업1 ████████░░░░░░░  작업2 ████████░░░░░░  → 총 16초
비동기: 작업1 ████████
       작업2 ████████
       작업3 ████████  → 총 8초 (동시에 대기)
```

```python
import asyncio

# 비동기 함수 (코루틴) 정의
async def fetch_data(name, delay):
    print(f"[{name}] 데이터 요청 중...")
    await asyncio.sleep(delay)  # 비동기 대기 (I/O 시뮬레이션)
    print(f"[{name}] 데이터 수신 완료! ({delay}초)")
    return f"{name}_data"

# 순차 실행 (총 6초)
async def sequential():
    result1 = await fetch_data("API1", 2)
    result2 = await fetch_data("API2", 3)
    return [result1, result2]

# 병렬 실행 (총 3초, 가장 느린 것만큼)
async def parallel():
    results = await asyncio.gather(
        fetch_data("API1", 2),
        fetch_data("API2", 3),
        fetch_data("API3", 1)
    )
    return results

asyncio.run(parallel())
```

        "https://jsonplaceholder.typicode.com/users/1",
        "https://jsonplaceholder.typicode.com/users/2",
        "https://jsonplaceholder.typicode.com/users/3",
    ]

    async with aiohttp.ClientSession() as session:
        tasks = [fetch_url(session, url) for url in urls]
        results = await asyncio.gather(*tasks)

## 2. 멀티스레딩과 멀티프로세싱

### 언제 무엇을 사용할까?

| 상황 | 권장 방법 | 이유 |
|------|---------|------|
| I/O 바운드 (API 호출, DB 쿼리) | asyncio 또는 ThreadPoolExecutor | GIL의 영향 최소 |
| CPU 바운드 (계산 집중) | ProcessPoolExecutor | GIL 우회, 병렬 처리 |
| 간단한 병렬 작업 | concurrent.futures | 추상화된 인터페이스 |

```python
from concurrent.futures import ThreadPoolExecutor, ProcessPoolExecutor
import time

# I/O 바운드 → 스레드 풀
def download_file(url):
    time.sleep(1)  # 네트워크 대기 시뮬레이션
    return f"Downloaded: {url}"

with ThreadPoolExecutor(max_workers=5) as executor:
    urls = ["http://example.com/" + str(i) for i in range(10)]
    results = list(executor.map(download_file, urls))

# CPU 바운드 → 프로세스 풀
def calculate_prime(n):
    """소수 판별 (CPU 집중 작업)"""
    if n < 2:
        return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return False
    return True

with ProcessPoolExecutor() as executor:
    numbers = range(1, 10000)
    primes = list(filter(None, executor.map(calculate_prime, numbers)))
```

## 4. 디자인 패턴

### 싱글톤 (Singleton)

```python
class DatabaseConnection:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance.connection = None
        return cls._instance

    def connect(self, url):
        if self.connection is None:
            self.connection = f"Connected to {url}"
        return self.connection

db1 = DatabaseConnection()
db2 = DatabaseConnection()
print(db1 is db2)  # True (같은 인스턴스)
```

### 옵저버 (Observer)

```python
class EventEmitter:
    def __init__(self):
        self._handlers: Dict[str, List] = {}

    def on(self, event: str, handler):
        if event not in self._handlers:
            self._handlers[event] = []
        self._handlers[event].append(handler)

    def emit(self, event: str, *args, **kwargs):
        for handler in self._handlers.get(event, []):
            handler(*args, **kwargs)

emitter = EventEmitter()
emitter.on('user_created', lambda user: print(f"새 사용자: {user}"))
emitter.on('user_created', lambda user: send_welcome_email(user))
emitter.emit('user_created', {'name': '홍길동', 'email': 'hong@example.com'})
```

## 5. 성능 최적화

```python
import timeit
from functools import lru_cache

# 메모이제이션으로 피보나치 최적화
@lru_cache(maxsize=None)
def fibonacci(n):
    if n < 2:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

# 실행 시간 측정
print(timeit.timeit(lambda: fibonacci(100), number=1000))

# 제너레이터로 메모리 절약
def read_large_file(filepath):
    with open(filepath) as f:
        for line in f:
            yield line.strip()

# 슬롯으로 메모리 절약
class Point:
    __slots__ = ['x', 'y']  # 딕셔너리 대신 고정 슬롯 사용

    def __init__(self, x, y):
        self.x = x
        self.y = y
```

## 핵심 포인트 정리

- asyncio는 I/O 바운드 작업에서 극적인 성능 향상을 제공합니다. API 서버에서 여러 외부 API를 동시에 호출할 때 특히 유용합니다
- Python의 GIL(Global Interpreter Lock) 때문에 CPU 집중 작업은 멀티스레딩이 아닌 멀티프로세싱을 사용해야 합니다
- `@lru_cache`는 한 줄로 메모이제이션을 구현할 수 있는 강력한 데코레이터입니다
- 비밀번호는 반드시 bcrypt 같은 단방향 해시 함수로 저장하고, 절대 평문으로 저장하지 마세요
