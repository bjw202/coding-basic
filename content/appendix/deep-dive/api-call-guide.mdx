# API 호출 실전가이드

## 학습 목표

이 자료를 마치면 다음을 할 수 있습니다:
- fetch와 axios로 외부 API를 호출할 수 있습니다
- 에러 처리 패턴을 실무에 적용할 수 있습니다
- 날씨 API, 공공데이터 API, AI API를 연동할 수 있습니다
- Express에서 외부 API 프록시를 구현할 수 있습니다

---

## 1. API 호출 기초

```
내 서버 (Express) ──────▶ 외부 API 서버
                  ◀──────

1. 내 서버에서 요청(Request) 보냄
2. 외부 API가 데이터로 응답(Response)
3. 받은 데이터를 가공해서 활용
```

### 서버 → 서버 vs 브라우저 → 서버

| 구분 | 서버 → 서버 | 브라우저 → 서버 |
|------|------------|---------------|
| CORS 제한 | 없음 | 있음 |
| API 키 노출 | 안전 | 위험 |
| 직접 호출 | 자유롭게 가능 | 제한적 |

> **중요**: API 키가 필요한 호출은 반드시 **서버(백엔드)**에서 해야 합니다.

## 2. fetch API (내장)

```javascript
// 기본 GET 요청
const response = await fetch('https://jsonplaceholder.typicode.com/users')
const users = await response.json()
console.log(users)

// POST 요청
const response = await fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    name: '홍길동',
    email: 'hong@example.com'
  })
})

const data = await response.json()

// 에러 처리 (fetch는 HTTP 에러를 throw하지 않음!)
if (!response.ok) {
  throw new Error(`HTTP 에러: ${response.status}`)
}
```

## 3. axios 라이브러리

```bash
npm install axios
```

```javascript
import axios from 'axios'

// 기본 설정 (베이스 URL, 헤더 등)
const api = axios.create({
  baseURL: 'https://api.example.com',
  timeout: 5000,
  headers: {
    'Content-Type': 'application/json'
  }
})

// 인증 토큰 자동 추가
api.interceptors.request.use(config => {
  const token = process.env.API_TOKEN
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 요청
const { data } = await api.get('/users')
const { data: newUser } = await api.post('/users', { name: '홍길동' })
const { data: updated } = await api.patch('/users/1', { name: '김철수' })
await api.delete('/users/1')
```

### fetch vs axios 비교

| 항목 | fetch | axios |
|------|-------|-------|
| 설치 | 불필요 (내장) | npm install 필요 |
| HTTP 에러 | throw 안 함 | throw 함 |
| 자동 JSON | `response.json()` 필요 | 자동 파싱 |
| 인터셉터 | 없음 | 있음 |
| 브라우저 지원 | 모던 브라우저 | 광범위 |

## 4. 에러 처리 패턴

```javascript
// try-catch 패턴
async function getUsers() {
  try {
    const { data } = await api.get('/users')
    return data
  } catch (error) {
    if (error.response) {
      // 서버가 응답했지만 에러 상태 코드
      console.error('서버 에러:', error.response.status, error.response.data)
    } else if (error.request) {
      // 요청은 보냈지만 응답이 없음
      console.error('응답 없음:', error.request)
    } else {
      // 요청 설정 중 에러
      console.error('에러:', error.message)
    }
    throw error
  }
}
```

## 5. 날씨 API 연동 (OpenWeatherMap)

```javascript
// .env
// OPENWEATHER_API_KEY=your-api-key

const getWeather = async (city) => {
  const url = `https://api.openweathermap.org/data/2.5/weather`
  const params = new URLSearchParams({
    q: city,
    appid: process.env.OPENWEATHER_API_KEY,
    units: 'metric',
    lang: 'kr'
  })

  const response = await fetch(`${url}?${params}`)
  if (!response.ok) throw new Error('날씨 정보를 가져올 수 없습니다')

  const data = await response.json()
  return {
    city: data.name,
    temp: data.main.temp,
    description: data.weather[0].description
  }
}
```

## 7. Express에서 외부 API 프록시

브라우저에서 직접 API 키를 쓰면 노출됩니다. 서버를 프록시로 사용하세요:

```javascript
// Express 프록시 엔드포인트
app.get('/api/weather/:city', async (req, res) => {
  try {
    const { city } = req.params
    const weatherData = await getWeather(city)
    res.json(weatherData)
  } catch (error) {
    res.status(500).json({ error: '날씨 정보를 가져올 수 없습니다' })
  }
})

// 브라우저에서는 우리 서버로만 호출 (API 키 노출 없음)
// fetch('/api/weather/Seoul')
```

## 핵심 포인트 정리

- API 키가 있는 호출은 반드시 서버에서만 합니다. 브라우저에서 직접 호출하면 API 키가 노출됩니다
- `fetch`는 HTTP 에러(400, 500 등)를 throw하지 않으므로 `response.ok`를 반드시 확인하세요
- `axios`는 설치가 필요하지만 자동 JSON 파싱, 인터셉터 등 편의 기능이 많습니다
- 외부 API 호출 시 항상 타임아웃을 설정하고 에러 처리를 구현하세요
