# MySQL 기초

## 학습 목표

이 자료를 마치면 다음을 할 수 있습니다:
- MySQL을 설치하고 접속할 수 있습니다
- 데이터베이스와 테이블을 생성하고 관리할 수 있습니다
- CRUD 기본 쿼리(SELECT, INSERT, UPDATE, DELETE)를 작성할 수 있습니다

## 1. MySQL 개요

MySQL은 "세계에서 가장 많이 사용되는 오픈소스 관계형 데이터베이스(RDBMS)"입니다.

### 왜 MySQL인가?

| 장점 | 설명 |
|------|------|
| 무료 & 오픈소스 | 커뮤니티 에디션 무료 사용 |
| 검증된 안정성 | 20년+ 역사, 전 세계 기업 사용 |
| 쉬운 학습 | SQL 표준에 가장 가까운 문법 |
| 높은 호환성 | 거의 모든 프로그래밍 언어 지원 |

### 관계형 DB 개념

```
데이터베이스 (Database)
  └── 테이블 (Table)
       ├── 컬럼 (Column) = 속성, 필드
       └── 로우 (Row) = 데이터, 레코드
```

## 3. 데이터베이스와 테이블

```sql
-- 데이터베이스 생성
CREATE DATABASE myapp CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE myapp;

-- 테이블 생성
CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  age INT,
  role ENUM('user', 'admin') DEFAULT 'user',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 테이블 구조 확인
DESCRIBE users;

-- 테이블 목록
SHOW TABLES;
```

## 5. CRUD 기본 쿼리

```sql
-- INSERT (생성)
INSERT INTO users (name, email, age) VALUES ('홍길동', 'hong@example.com', 30);

-- SELECT (조회)
SELECT * FROM users;
SELECT id, name, email FROM users;
SELECT * FROM users WHERE age > 25;
SELECT * FROM users ORDER BY created_at DESC LIMIT 10;

-- UPDATE (수정)
UPDATE users SET age = 31, role = 'admin' WHERE id = 1;

-- DELETE (삭제)
DELETE FROM users WHERE id = 1;

-- 소프트 삭제 패턴 (실무 권장)
ALTER TABLE users ADD COLUMN deleted_at TIMESTAMP NULL;
UPDATE users SET deleted_at = NOW() WHERE id = 1;
SELECT * FROM users WHERE deleted_at IS NULL;  -- 삭제되지 않은 데이터만 조회
```

## 6. 조건과 정렬

```sql
-- WHERE 조건
SELECT * FROM users WHERE role = 'admin' AND age >= 20;
SELECT * FROM users WHERE name LIKE '김%';  -- '김'으로 시작
SELECT * FROM users WHERE age BETWEEN 20 AND 30;
SELECT * FROM users WHERE id IN (1, 2, 3);
SELECT * FROM users WHERE email IS NOT NULL;

-- ORDER BY
SELECT * FROM users ORDER BY created_at DESC;
SELECT * FROM users ORDER BY age ASC, name DESC;

-- LIMIT & OFFSET (페이지네이션)
SELECT * FROM users ORDER BY id LIMIT 10 OFFSET 20;  -- 3페이지
```

## 7. 집계 함수와 그룹화

```sql
-- 집계 함수
SELECT COUNT(*) FROM users;
SELECT AVG(age), MAX(age), MIN(age) FROM users;
SELECT SUM(price) FROM orders;

-- GROUP BY
SELECT role, COUNT(*) as count FROM users GROUP BY role;

-- HAVING (GROUP BY 후 필터)
SELECT category, AVG(price) as avg_price
FROM products
GROUP BY category
HAVING avg_price > 50000;
```

## 8. JOIN - 테이블 연결

```sql
-- 사용자와 주문 테이블
CREATE TABLE orders (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  total_price DECIMAL(10, 2) NOT NULL,
  status ENUM('pending', 'paid', 'shipped') DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- INNER JOIN (양쪽 모두 있을 때만)
SELECT u.name, o.total_price, o.status
FROM users u
INNER JOIN orders o ON u.id = o.user_id;

-- LEFT JOIN (왼쪽 기준, 오른쪽에 없으면 NULL)
SELECT u.name, COUNT(o.id) as order_count
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
GROUP BY u.id, u.name;
```

## 핵심 포인트 정리

- `AUTO_INCREMENT`와 `PRIMARY KEY`를 모든 테이블에 기본으로 설정하세요
- 실무에서는 DELETE 대신 `deleted_at TIMESTAMP` 컬럼으로 소프트 삭제를 많이 사용합니다
- `JOIN`은 처음엔 헷갈리지만 `INNER JOIN`(교집합)과 `LEFT JOIN`(왼쪽 기준) 두 가지만 익혀도 대부분의 쿼리를 작성할 수 있습니다
- `EXPLAIN`으로 쿼리 실행 계획을 확인하면 느린 쿼리를 최적화할 수 있습니다
