# Claude Code 필수 명령어

## 학습 목표

이 자료를 마치면 다음을 할 수 있습니다:
- Claude Code를 설치하고 프로젝트에서 바로 사용할 수 있습니다
- 코드 탐색, 수정, 테스트, 커밋까지 터미널에서 한 번에 처리할 수 있습니다
- 범위를 제한하고 안전하게 AI 코딩을 운영할 수 있습니다

---

## Claude Code란?

Claude Code는 프로젝트 전체 코드를 이해하고 함께 수정해주는 AI CLI 도구입니다. 자연어(영어 권장)로 지시하고, 설명은 한국어로 이해하는 방식이 가장 안정적입니다.

---

## 1. 설치 및 기본 설정

### 설치

```bash
npm install -g @anthropic-ai/claude-code
```

### 버전 확인

```bash
claude --version
```

### 로그인 / API 키 설정

```bash
claude auth login
```

또는 환경변수로 설정:

```bash
export ANTHROPIC_API_KEY=sk-xxxx
```

---

## 2. 프로젝트에서 기본 사용 흐름

```bash
cd 프로젝트폴더
claude
```

이후부터는 대화형 세션에서 자연어로 계속 지시합니다.

---

## 3. 가장 많이 쓰는 핵심 명령 (의도별)

### 프로젝트 / 코드 설명 요청

```
이 프로젝트 구조를 설명해줘
(explain this project)

이 파일이 무슨 역할인지 설명해줘
(explain src/server.ts)
```

### 코드 수정 (바이브코딩 핵심)

```
이 코드를 더 읽기 쉽게 리팩토링해줘
(refactor this file to be more readable)

이 함수에 에러 처리를 추가해줘
(add error handling to this function)

콜백을 async/await 구조로 바꿔줘
(convert this to async/await)
```

### 기능 추가 요청

```
JWT 기반 로그인 API를 추가해줘
(add login API using JWT)

이 API에 페이지네이션을 추가해줘
(add pagination to this endpoint)

zod로 입력값 검증을 추가해줘
(add validation with zod)
```

---

## 4. 범위 제한 (사고 방지 필수)

Claude Code가 의도하지 않은 파일을 수정하지 않도록 범위를 명시합니다.

```
src/api 폴더만 보고 작업해줘
(look only at src/api)

src 폴더 안의 파일만 수정해줘
(only modify files under src/)

node_modules, dist는 무시해줘
(ignore node_modules and dist)
```

> **중요**: 범위 지정 없는 수정 요청은 의도하지 않은 대규모 변경을 유발할 수 있습니다. 항상 범위를 먼저 지정하세요.

---

## 5. 테스트 / 디버깅 활용

```
이 파일에 대한 유닛 테스트를 작성해줘
(write unit tests for this file)

실패한 테스트를 고쳐줘
(fix failing tests)

아래 에러 로그를 분석해줘
(analyze this error log)
```

에러 로그를 붙일 때 패턴:

```
아래는 에러 로그야:
[로그 붙여넣기]
```

---

## 6. Git 연계 (실무에서 매우 유용)

```
변경된 내용을 요약해줘
(summarize changes)

컨벤션에 맞는 커밋 메시지를 만들어줘
(generate a conventional commit message)

PR 설명을 작성해줘
(write a PR description)
```

---

## 7. 잘 먹히는 명령 스타일

```
설명은 하지 말고 코드만 수정해줘
(do not explain, just modify the code)

변경은 최소한으로 해줘
(keep changes minimal)

기존 코드 스타일을 유지해줘
(follow existing code style)
```

---

## 8. 여러 파일 동시에 작업할 때

```
UserService가 사용되는 모든 곳을 찾아줘
(search for all usages of UserService)

관련된 모든 참조를 함께 수정해줘
(update all references accordingly)
```

---

## 9. Claude Code + 터미널 실전 패턴

테스트가 실패했을 때:

```bash
npm test
# (실패 로그 복사)
```

Claude Code에 붙여넣기:

```
이 테스트 실패 로그를 기준으로 문제를 해결해줘
```

---

## 10. 바이브코딩 추천 워크플로우

1. `claude` 실행
2. "이 프로젝트를 설명해줘" 로 컨텍스트 파악
3. 원하는 기능을 자연어로 명확히 지시
4. "변경은 최소한으로 해줘" 를 함께 사용
5. 테스트 실행으로 검증
6. 커밋 메시지 생성 후 직접 `git commit`

---

## 11. 자주 쓰는 치트 문장 (복붙용)

| 목적 | 프롬프트 |
|------|---------|
| 단계적 사고 | 차근차근 생각해서 처리해줘 (think step by step) |
| 운영 환경 기준 | 운영 환경을 기준으로 고려해줘 (assume production environment) |
| 엣지 케이스 | 엣지 케이스도 고려해줘 (consider edge cases) |
| 가독성 우선 | 가독성을 최우선으로 해줘 (optimize for readability) |

---

## 주의사항 (실무 중요)

- 큰 변경은 반드시 범위 제한부터 설정한 후 진행합니다
- 자동 수정 후 `git diff`로 직접 확인합니다
- 한 번에 많은 요구는 금물 — 기능 단위로 요청합니다

---

## 핵심 포인트 정리

- Claude Code는 자동 코딩기가 아니라 같이 일하는 시니어 개발자입니다
- 지시는 명확하게, 범위는 좁게, 검증은 사람이 해야 합니다
- `explain → 지시 → 범위 제한 → 테스트 → 커밋` 흐름을 습관으로 만드세요
- 에러 로그가 있을 때는 로그를 그대로 붙여서 분석을 요청하면 매우 효과적입니다
