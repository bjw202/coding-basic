# CLAUDE.md 작성법

## 학습 목표

이 자료를 마치면 다음을 할 수 있습니다:
- CLAUDE.md가 무엇이고 왜 필요한지 이해할 수 있습니다
- 프로젝트에 맞는 AI 코딩 규칙을 작성할 수 있습니다
- API 계약 보호, 보안 규칙, 테스트 원칙을 명문화할 수 있습니다

---

## CLAUDE.md란?

CLAUDE.md는 Claude Code / Cursor / Copilot 등 AI 코딩 도구가 이 레포지토리에서 따라야 할 절대 규칙을 정의하는 파일입니다.

**목적**: 빠르게 개발하되, 사고 없이 · API 계약을 깨지 않고 · 운영 안정성을 유지하는 것.

---

## 1. 기본 원칙 (절대 규칙)

좋은 CLAUDE.md는 다음 4가지 원칙을 담아야 합니다:

- AI는 보조 도구이며, 최종 책임은 사람에게 있다
- 범위 지정 없는 수정은 금지한다
- 한 번의 요청은 하나의 목적만 가진다
- 기존 동작 / API 계약 / 데이터 구조를 암묵적으로 변경하지 않는다

---

## 2. 작업 범위 규칙

```markdown
## 작업 범위 규칙

AI는 아래 규칙을 반드시 따른다.

- 수정 범위는 명시된 디렉토리/파일만 허용된다
- 명시되지 않은 파일은 읽기만 가능하다
- `node_modules`, `dist`, `build` 디렉토리는 항상 무시한다
```

---

## 3. API 계약 규칙 (가장 중요)

```markdown
## API 계약 규칙

- 기존 API 요청/응답 스키마는 절대 변경 금지
- 변경이 필요한 경우:
  - 새로운 버전(v2)으로 분리
  - 또는 하위 호환성을 유지

AI에게 반드시 명시할 문장:
"기존 API 응답 스키마는 절대 변경하지 말 것."
```

> **왜 이것이 가장 중요한가?** API 응답 구조가 변경되면 프론트엔드 전체가 장애 상태가 됩니다. 실제로 "응답 구조를 더 직관적으로 바꿔줘"라는 한 마디로 배포 직후 흰 화면 장애가 발생한 사례가 있습니다.

---

## 4. 에러 처리 규칙

```markdown
## 에러 처리 규칙

- 모든 실패는 의미 있는 HTTP 상태 코드를 사용한다
- 성공/실패를 200 하나로 처리하지 않는다
- 에러 응답 형식은 아래로 통일한다:

{ "error": { "code": "ERROR_CODE", "message": "human readable message" } }
```

---

## 5. 인증 / 보안 규칙

```markdown
## 인증 / 보안 규칙

- 인증/인가 로직은 AI 단독 수정 금지
- 토큰, 비밀번호, 개인정보는:
  - 로그에 남기지 않는다
  - 코드에 하드코딩하지 않는다
```

---

## 6. 로깅 / 관측성 규칙

```markdown
## 로깅 규칙

- 운영 로그 제거 금지
- 에러 로그에는 최소한 아래 정보 포함:
  - requestId
  - 사용자 식별자(가능한 경우)
  - 핵심 파라미터(민감 정보 제외)
```

---

## 7. 테스트 규칙

```markdown
## 테스트 규칙

- 로직 변경 시 테스트 없는 수정 금지
- 버그 수정은 반드시 재현 테스트 → 수정 → 통과 순서로 진행
- 테스트가 불가능한 경우, 그 이유를 명시한다
```

---

## 8. Git / 협업 규칙

```markdown
## Git / 협업 규칙

- AI가 수정한 PR은 제목에 [AI]를 붙인다
- PR은 1 기능 = 1 PR 원칙을 따른다
- 대규모 변경은 단계적으로 PR을 분리한다
```

## 9. 금지 프롬프트 예시

```markdown
## 금지 프롬프트 (사고 가능성 높음)

아래와 같은 요청은 사용을 금지한다:

- "전체 코드 정리해줘"
- "더 깔끔하게 만들어줘"
- "알아서 최적화해줘"
```

---

## 10. 권장 프롬프트 템플릿

```markdown
## 권장 프롬프트 템플릿

[범위]
src/orders 디렉토리만 수정한다.

[목적]
POST /orders API에서 중복 생성 문제를 해결한다.

[제약]
- 기존 응답 스키마 유지
- 멱등성 보장
- 로그/보안 로직 변경 금지

[검증]
- 기존 테스트 통과
- 중복 요청 재현 테스트 추가
```

## CLAUDE.md 파일 위치

프로젝트 루트에 `CLAUDE.md` 파일로 저장합니다:

```
my-project/
├── CLAUDE.md        ← AI 코딩 규칙 파일
├── src/
├── package.json
└── ...
```

Claude Code는 실행 시 이 파일을 자동으로 읽어 규칙을 적용합니다.

---

## 핵심 포인트 정리

- CLAUDE.md는 AI 도구에게 주는 팀 계약서입니다
- 이 파일을 읽지 않고 수정하는 AI 출력은 리뷰 없이 병합되지 않아야 합니다
- API 계약 규칙이 가장 중요합니다. 이것만 지켜도 대형 장애의 70%를 막을 수 있습니다
- 이 레포지토리에서 AI는 빠른 손이지만, 방향과 책임은 사람이 가집니다
