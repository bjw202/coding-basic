# 터미널과 IT 기초

## 학습 목표

이 페이지를 마치면 다음을 할 수 있습니다.

- 터미널에서 핵심 명령어를 자신 있게 실행할 수 있습니다.
- 개발자들이 당연히 안다고 가정하는 기초 IT 용어를 설명할 수 있습니다.
- AI에게 정확한 용어로 지시하여 더 좋은 코드를 얻을 수 있습니다.

## 핵심 터미널 명령어

### 명령어 치트시트 (Mac/Linux 기준)

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `pwd` | 현재 위치 확인 | `pwd` → `/Users/내이름/Documents` |
| `ls` | 목록 보기 | `ls -la` (숨김 파일 포함 상세 보기) |
| `cd` | 폴더 이동 | `cd my-project` / `cd ..` / `cd ~` |
| `mkdir` | 폴더 만들기 | `mkdir my-project` |
| `touch` | 파일 만들기 (Mac) | `touch index.js` |
| `rm` | 파일/폴더 삭제 | `rm file.txt` / `rm -rf folder/` |
| `cp` | 파일 복사 | `cp a.txt b.txt` |
| `mv` | 파일 이동/이름 변경 | `mv old.txt new.txt` |
| `cat` | 파일 내용 보기 | `cat index.js` |
| `clear` | 화면 정리 | `clear` |

### 자주 쓰는 개발자 명령어

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `node -v` | Node.js 버전 확인 | `v20.11.0` |
| `npm -v` | npm 버전 확인 | `10.2.4` |
| `git --version` | Git 버전 확인 | `git version 2.39.3` |
| `tail -f` | 실시간 로그 보기 | `tail -f log.txt` |
| `grep` | 내용 검색 | `grep "error" log.txt` |
| `ping` | 네트워크 연결 확인 | `ping google.com` |

### Windows(cmd) vs Mac/Linux 차이

| 동작 | Mac/Linux | Windows (cmd) |
|------|-----------|----------------|
| 목록 보기 | `ls` | `dir` |
| 현재 위치 | `pwd` | `cd` (인자 없이) |
| 화면 정리 | `clear` | `cls` |
| 파일 삭제 | `rm` | `del` |
| 폴더 삭제 | `rm -rf` | `rmdir /s /q` |

> Windows 사용자는 Git Bash 또는 WSL을 설치하면 Mac과 동일하게 사용할 수 있습니다.

### 터미널 꿀팁

1. **Tab 자동완성**: 폴더명 일부만 치고 Tab → 자동완성 (`cd Doc` + Tab → `cd Documents`)
2. **위/아래 화살표**: 이전에 입력한 명령어 불러오기
3. **Ctrl + C**: 실행 중인 명령 강제 종료 ("아 이거 잘못됐다" 싶을 때)
4. **Ctrl + L**: clear와 같음 (화면 정리)

### 주의할 명령어

```bash
# 절대 실행하지 마세요!
rm -rf /        # 시스템 전체 삭제
rm -rf *        # 현재 폴더의 모든 파일 삭제

# rm -rf 실행 전 항상 pwd로 위치 확인!
```

## IT 기초 용어 사전

AI에게 정확한 용어로 지시할수록 더 좋은 결과를 얻습니다. 바이브코딩 수업에서 자주 등장하는 핵심 용어만 정리했습니다.

### 웹 개발 기초

| 용어 | 설명 |
|------|------|
| **프론트엔드 (Frontend)** | 사용자가 직접 보는 화면. HTML, CSS, JavaScript, React 등 |
| **백엔드 (Backend)** | 데이터 처리, 인증, 비즈니스 로직. Node.js, Python 등 |
| **풀스택 (Fullstack)** | 프론트엔드 + 백엔드 모두 개발 |
| **API** | 프론트엔드와 백엔드가 대화하는 창구. "API를 호출한다" = 서버에 요청을 보낸다 |
| **REST API** | HTTP 메서드(GET, POST 등)로 자원을 다루는 API 설계 방식 |

### 네트워크 기초

| 용어 | 설명 |
|------|------|
| **HTTP/HTTPS** | 웹에서 데이터를 주고받는 규칙. HTTPS는 암호화된 HTTP |
| **요청 (Request) / 응답 (Response)** | 클라이언트가 요청 → 서버가 응답 |
| **GET/POST/PUT/DELETE** | HTTP 메서드. 조회/생성/수정/삭제 |
| **상태 코드 (Status Code)** | HTTP 응답 결과 숫자. 200(성공), 404(없음), 500(서버 에러) |
| **localhost** | 내 컴퓨터 자신을 가리키는 주소. 127.0.0.1 |
| **포트 (Port)** | 하나의 컴퓨터에서 프로그램을 구분하는 번호. 웹 서버는 보통 3000, 8080 등 |
| **URL** | 웹 자원의 전체 주소. `https://vibeclass.kr/courses/1` |

### 데이터베이스 기초

| 용어 | 설명 |
|------|------|
| **데이터베이스 (DB)** | 데이터를 저장하는 시스템. MySQL, PostgreSQL, MongoDB 등 |
| **SQL** | 데이터베이스를 다루는 언어. SELECT, INSERT, UPDATE, DELETE |
| **테이블 (Table)** | 데이터를 행과 열로 정리한 구조. 엑셀 시트와 비슷 |
| **ORM** | 코드로 DB를 다루는 도구. Prisma, Sequelize 등 |
| **NoSQL** | 테이블 구조가 아닌 유연한 데이터 저장. MongoDB, Redis 등 |

### 개발 도구 기초

| 용어 | 설명 |
|------|------|
| **IDE** | 코드 작성, 디버깅, 실행을 한 곳에서. VSCode, Cursor 등 |
| **터미널 (Terminal)** | 명령어로 컴퓨터를 조작하는 도구 |
| **Git** | 코드 변경 이력을 관리하는 버전 관리 도구 |
| **GitHub** | Git 저장소를 호스팅하는 서비스 |
| **npm** | Node.js 패키지 매니저. 외부 라이브러리 설치/관리 |
| **package.json** | 프로젝트 설정 파일. 의존성, 스크립트 정의 |
| **.env** | 환경변수를 저장하는 파일. API 키 등 민감한 정보 보관 |

### 바이브코딩에서 자주 나오는 용어

| 용어 | 설명 |
|------|------|
| **컴포넌트 (Component)** | 재사용 가능한 UI 조각. 버튼, 카드, 헤더 등 |
| **상태 (State)** | 컴포넌트가 가지고 있는 데이터. 변경되면 화면이 다시 그려짐 |
| **렌더링 (Rendering)** | 코드를 화면에 그리는 과정 |
| **비동기 (Asynchronous)** | 순서대로 기다리지 않고 실행. async/await 패턴 |
| **배포 (Deploy)** | 만든 프로그램을 서버에 올려 실제 서비스하는 것 |
| **환경변수 (Environment Variable)** | 코드에 직접 적지 않는 설정값. 비밀번호, API 키 등 |
| **의존성 (Dependency)** | 프로젝트가 필요로 하는 외부 라이브러리 |
| **프레임워크 (Framework)** | 구조를 강제하는 개발 틀. React, Next.js, Express 등 |

### AI/프롬프트 관련 용어

| 용어 | 설명 |
|------|------|
| **프롬프트 (Prompt)** | AI에게 내리는 지시문 |
| **컨텍스트 (Context)** | AI가 이해해야 할 배경 정보. 많을수록 정확한 답변 |
| **할루시네이션 (Hallucination)** | AI가 없는 정보를 만들어내는 현상. 팩트체크 필요 |
| **토큰 (Token)** | AI가 처리하는 텍스트 단위. 한글은 1글자 = 약 2토큰 |
| **프롬프트 엔지니어링** | 효과적인 프롬프트 작성 기술. AI 활용의 핵심 |

## 핵심 포인트 정리

### 터미널 명령어 요약

```
기본 이동: ls  cd  pwd  mkdir  clear
파일 작업: touch  cp  mv  rm
개발 확인: node -v  npm -v  git --version
로그 확인: tail -f  grep
```

### 용어 활용 팁

- 모르는 용어가 나오면 AI에게 "이 용어가 뭐야?"라고 물어봐도 됩니다.
- AI에게 지시할 때 정확한 용어를 사용할수록 결과물이 좋아집니다.
- 처음에 모두 외울 필요 없습니다. 이 페이지를 북마크해두고 필요할 때 찾아보세요.
