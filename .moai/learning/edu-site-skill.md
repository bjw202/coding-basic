# moai-domain-edu-site 스킬 가이드

> 이 문서는 교육용 Nextra 문서 사이트를 반복 생성할 때 MoAI를 최적으로 활용하는 방법을 정리합니다.

---

## 스킬 개요

| 항목 | 내용 |
|------|------|
| 스킬 파일 | `.claude/skills/moai/moai-domain-edu-site.md` |
| 버전 | 1.0.0 |
| 목적 | 교육용 Nextra MDX 콘텐츠 사이트 생성 도메인 지식 제공 |
| 자동 트리거 | `nextra`, `교육`, `세션`, `학습`, `강의`, `curriculum`, `bootcamp` 키워드 |

### 스킬이 하는 일

`/moai plan` 명령에 위 키워드가 포함되면 MoAI와 서브에이전트들이 이 스킬을 자동으로 로드합니다.
로드된 스킬은 다음 지식을 에이전트에게 주입합니다:

- MDX 세션 표준 섹션 구조 (학습목표, 전체그림, 핵심개념, 실습, 요약)
- Mermaid 안전 문법 규칙 (아포스트로피 금지, `+` 사용 제한)
- 콘텐츠 품질 기준 (왜/맥락/연결포인트/흔한오해)
- SPEC 배치 분할 전략 (3~4 세션/SPEC)
- 커리큘럼 명세 파일 포맷

---

## 새 프로젝트 시작 순서

### Step 0: 커리큘럼 명세 파일 작성 (사람이 직접)

MoAI에게 시키기 전에 아래 포맷으로 `curriculum.md` 파일을 먼저 만듭니다.
이 파일이 MoAI의 모든 후속 작업의 기준이 됩니다.

```markdown
# [과정명] 커리큘럼 명세

## 대상 독자
[예: 프로그래밍 경험 없는 한국어 사용 주니어 개발자]

## 기술 스택
[예: Next.js 15, Nextra 4.x, TypeScript, 한국어]

## 콘텐츠 구조
| 주차 | 세션 | 제목         | 핵심 개념 3개                  | 학습 목표       |
|------|------|--------------|-------------------------------|----------------|
| W1   | S1   | 개발환경 세팅 | VSCode, Git, .env             | 환경 구성 완료 |
| W1   | S2   | FE/BE 기초   | Node.js, Express, 모듈        | API 이해       |
| ...  | ...  | ...           | ...                           | ...            |

## 품질 기준
- 각 핵심 개념마다 "왜 필요한가?" 설명 포함
- 세션당 Mermaid 빅픽처 다이어그램 1개
- 미래 세션 연결 포인트 명시
- 흔한 오해 교정 섹션 포함
- 멘토 어조 (친근하고 존중하는 표현)
```

---

### Step 1: 프로젝트 초기화 (최초 1회)

```
/moai project
```

`product.md`, `structure.md`, `tech.md`를 생성하여 MoAI가 프로젝트를 이해하도록 합니다.

---

### Step 2: 인프라 SPEC 생성 및 실행

```
/moai plan "Nextra 4.x 교육 사이트 인프라 — [과정명]
커리큘럼 파일: curriculum.md 참고
구조: W1~W4, 각 3세션
표준 MDX 섹션: 학습목표, 핵심개념, 빅픽처다이어그램, 실습, 요약"
```

SPEC 승인 후:

```
/moai run SPEC-INFRA-XXX
```

생성되는 것: `content/` 폴더 구조, `_meta.js` 네비게이션, Next.js/Nextra 설정

---

### Step 3: 콘텐츠 SPEC 생성 및 실행 (배치 단위)

**중요**: 한 SPEC에 세션 3~4개가 최적입니다 (컨텍스트 제한).

```
/moai plan "콘텐츠 생성 — 1주차 S1~S3
커리큘럼 파일: curriculum.md W1 섹션 기반
품질 기준 (moai-domain-edu-site 스킬):
- 왜/맥락 설명을 생성 시점부터 포함 (생성 후 별도 보강 금지)
- Mermaid 안전 문법만 사용 (아포스트로피 금지, stateDiagram-v2에서 + 금지)
- 파일럿 1개 먼저 생성 후 승인하면 나머지 병렬 생성"
```

W2, W3, W4도 동일하게 반복:

```
/moai run SPEC-CONTENT-W1
/moai run SPEC-CONTENT-W2
...
```

---

### Step 4: Mermaid 에러 자동 수정

브라우저 콘솔에 Mermaid 파싱 에러가 보이면:

```
/moai loop [에러 메시지를 그대로 붙여넣기]
```

MoAI가 모든 MDX 파일에서 동일 패턴을 찾아 일괄 수정합니다.

## 핵심 Mermaid 문법 규칙

### 허용되는 다이어그램 타입

| 타입 | 문법 | 용도 |
|------|------|------|
| 탑다운 플로우차트 | `graph TD` | 개념 흐름, 아키텍처 |
| 좌우 플로우차트 | `graph LR` | 파이프라인, 레이어 |
| 시퀀스 다이어그램 | `sequenceDiagram` | API 흐름, 인증 흐름 |
| 상태 다이어그램 | `stateDiagram-v2` | 컴포넌트 생명주기 |
| ER 다이어그램 | `erDiagram` | 데이터베이스 관계 |

### 금지 문자 및 대안

| 금지 | 이유 | 대안 |
|------|------|------|
| `'` (아포스트로피) | NEWLINE 파싱 에러 | 제거하거나 생략 |
| `Let's Encrypt` | 아포스트로피 포함 | `Lets Encrypt` |
| `'use client'` 노드 라벨 시작 | 문자열 파서 오작동 | `use client` (따옴표 제거) |
| `stateDiagram-v2`에서 `+` | INVALID 토큰 에러 | `and`로 대체 |

## MDX 세션 표준 구조

각 세션 MDX 파일은 아래 섹션 순서를 따릅니다:

```
# N회차: [제목]

## 학습 목표       ← 3개 bullet
## 이번 세션 전체 그림  ← Mermaid 빅픽처 1개
## 핵심 개념       ← 왜필요한가? + 연결포인트 포함
## 흔한 오해       ← 최소 1개
## 실습            ← 기본 + 도전
## 요약            ← 용어 정리
```

---

## 세션당 품질 체크리스트

콘텐츠 생성 후 자동 검증 기준 (MoAI의 manager-quality가 사용):

- [ ] "왜 필요한가?" 블록쿼트 최소 3개
- [ ] "📎 연결 포인트" 콜아웃 최소 2개
- [ ] "흔한 오해" 섹션 최소 1개
- [ ] Mermaid 빅픽처 다이어그램 정확히 1개
- [ ] Mermaid 블록에 아포스트로피(`'`) 없음
- [ ] `stateDiagram-v2` 전환 라벨에 `+` 없음
- [ ] 모든 내용 한국어 (기술용어는 영어 병기)
- [ ] JSX 컴포넌트 `<Callout>` 미사용 — 블록쿼트 `>` 사용

## SPEC 네이밍 컨벤션

| SPEC ID 패턴 | 용도 | 예시 |
|-------------|------|------|
| `SPEC-INFRA-[SLUG]` | Nextra 기술 인프라 | `SPEC-INFRA-BOOTCAMP` |
| `SPEC-CONTENT-W[N]` | 주차별 콘텐츠 생성 | `SPEC-CONTENT-W1` |
| `SPEC-ENHANCE-[SLUG]` | 횡단 관심사 보강 | `SPEC-ENHANCE-WHY` |

## 이 프로젝트에서 배운 교훈

| 이번에 한 방식 (비효율) | 다음부터 할 방식 (최적) |
|----------------------|----------------------|
| 콘텐츠 생성 후 SPEC-ENHANCE로 따로 보강 | 생성 SPEC에 왜/맥락 기준을 처음부터 포함 |
| Mermaid 에러 사후 수동 수정 | SPEC에 금지 문자 규칙 명시 → 사전 방지 |
| 주차별 SPEC 순차 실행 | 파일럿 1개 승인 후 나머지 병렬 실행 |
